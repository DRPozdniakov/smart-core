Smart Core: Multi-User Collaboration Platform

Validation Summary — Infrastructure for Business Documentation Collaboration | February 12, 2026

Executive Summary

Smart Core is a MULTI-USER COLLABORATION PLATFORM enabling multiple Claude Code accounts to work simultaneously on shared business documentation — with full transparency for business owners.

Primary Purpose: Allow CTO, CFO, CEO, contractors, and advisors to collaborate on business docs (pitch deck, financial model, PRD, business plan) through a shared Neo4j knowledge graph. Single source of truth + real-time transparency = no email chains, no version conflicts, no manual sync.

Performance Validation: Three experiments confirm Smart Core delivers equal quality with 62.5% fewer tool calls and 64-70% token savings — while preventing stale data and contradictions that cost hours of manual audits. These efficiency gains are byproducts of the collaboration infrastructure, not the primary goal.

  • Revenue build-up sections (all years 2027-2030)

Experiment 3: Consistency Validation — Real-World Edge Cases

Purpose: Validate Smart Core's ability to catch cross-document inconsistencies (essential for multi-user environments where different contributors might introduce conflicting data).

Method: Documented 4 actual incidents where Smart Core would have prevented issues

Case 1: Stale Artifacts After Vision Changes

Incident: CTO made strategic shifts (pricing model, Seed phase refocus, team structure). Claude updated primary docs but old values survived in 8 secondary documents.

Stale Artifact Example How Long It Survived
Platform pricing model €8-25K/robot/year (old) vs €120K/enterprise/year (new) Multiple sessions
Payback period "4-month" (old) vs "<3 to ~21 months" (new) Multiple sessions
Market data $780M labeled 2025 (wrong) vs $940M (correct) Since creation
Seed timing Q3 2026 (old) vs Q4 2026 (new) Multiple sessions

Why It Happened (Without Smart Core)

  • Claude edits file-by-file, doesn't scan all 21 documents after each change
  • No system tracks which files contain which entities
  • Audits are expensive (hours of reading ~15,000 lines)
  • Contradictions invisible without manual cross-referencing

How Smart Core Prevents This

  • knowledge_call() returns ALL docs with entity + current values
  • Claude edits ALL affected files in one pass
  • merge_report() flags any remaining drift
  • synchronize_project() catches stragglers automatically

Case 2: Conflicting Entity Values

Incident: Same entity had different values in different documents — neither Claude nor user noticed for weeks.

Payback period 4 months (TAM-SAM-SOM) <3 to ~21 months (Exec Summary) <3 to ~21 months
Smart Core prevents contradictions at write time, not audit time. When the graph stores entity values, any new document that introduces a different value triggers an immediate conflict check. The cost of catching a contradiction at creation is near zero; the cost of catching it months later during investor due diligence could be the deal.

Key Insights

1. Change Propagation, Not Just Search

Search finds data. Propagation tracks relationships. When robot price changes, Smart Core knows which documents contain that entity, which sections calculate derived values, and which documents are now stale.

2. Quality Converges, Speed Diverges

Both approaches score ~90%+ quality (Smart Core 94%, Pure Claude 92% — statistical tie). But Smart Core is 400× faster. The graph doesn't make answers better — it makes them instant.

3. Silent Accumulation of Stale Data

10 stale values across 8 files survived for weeks because no one audited after each change (too expensive). Smart Core makes invisible problems visible with synchronize_project() at session start.

4. Neither Tool Alone Is Sufficient

Smart Core missed 3 files (edge folders, entity extraction gaps). Grep missed derived values (€400K fleet cost, revenue cascades). Optimal: Smart Core for relationships, Grep for completeness.
